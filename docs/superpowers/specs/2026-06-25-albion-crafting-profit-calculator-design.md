# 阿尔比恩装备制造利润计算器设计

## 背景

本项目是一个本地使用的 Albion Online 装备制造利润计算器 Web 应用。第一版聚焦单个装备的制造利润测算：用户选择一件装备，应用根据本地配方、Asia 市场价、制造返还率和会员销售税规则，计算单件利润、利润率和保本售价。

第一版不做城市对比、批量扫利润、精炼、账号系统、云同步或公开部署。

## 目标用户体验

打开页面后直接进入计算器，不做营销首页。用户完成一次计算的流程是：

1. 搜索并选择装备。
2. 查看装备图标、中文名、物品 ID、tier、附魔等级和配方材料。
3. 点击拉取 Asia 市场价，自动填入成品售价和材料价格。
4. 根据需要手动覆盖任意成品或材料价格。
5. 选择制造城市、制造站类型、是否使用专注。
6. 输入制造手续费；默认值为 `0`，用户可按实际制造站费用填写。
7. 勾选是否会员，系统展示对应销售税率。
8. 查看材料成本、返还折算、制造手续费、销售税、净收入、单件利润、利润率和保本售价。

## 技术路线

采用纯前端本地 Web：

- Vite + React + TypeScript 构建单页应用。
- 本地 JSON 保存装备和配方数据。
- 浏览器直接请求 Albion Online Data Project 的 Asia API 获取价格。
- 用户偏好和价格覆盖保存到 `localStorage`。
- 预留 `marketProvider` 抽象；如果后续遇到 API 跨域、缓存或稳定性问题，再增加本地 Node 代理。

## 页面结构

### 装备选择区

- 中文搜索装备名。
- 支持按装备类型、tier `T4-T8`、附魔等级 `.0-.4` 过滤。
- 搜索结果展示装备图标、中文名、英文名或物品 ID、tier、附魔等级。
- 选中装备后展示配方材料列表和当前品质。

### 市场价格区

- 一键拉取当前装备和材料的 Asia 市场价。
- 成品售价和材料单价都进入可编辑输入框。
- 自动价格只是默认值，用户的手动覆盖优先。
- 每个价格字段展示数据时间；无数据时标记“无市场数据”并提示手填。

### 制造参数区

- 选择制造城市。
- 选择制造站类型。
- 选择是否使用专注。
- 系统根据城市、制造站和专注状态给出默认返还率。
- 返还率允许手动覆盖。
- 制造手续费提供独立数字输入框，默认 `0`；该费用不从市场 API 自动获取。
- 提供“会员”开关。开启时使用会员销售税率，关闭时使用非会员销售税率。
- 销售税率只展示，不提供输入框。

### 利润结果区

- 材料毛成本。
- 返还材料价值。
- 返还后成本。
- 销售税。
- 净收入。
- 单件利润。
- 利润率。
- 保本售价。
- 明细表展示每种材料的数量、单价、成本和返还折算。

## 数据设计

### 装备和配方数据

本地静态数据包含：

- `itemId`：游戏物品 ID，例如 `T6_2H_AXE_AVALON@3`。
- `nameZh`：中文名。
- `nameEn`：英文名。
- `category`：装备类型。
- `tier`：物品阶级。
- `enchantment`：附魔等级。
- `craftAmount`：单次制造产出数量。
- `ingredients`：材料数组，包含材料物品 ID、中文名、数量。

第一版目标是尽量覆盖完整装备制造配方。数据导入逻辑应独立于应用运行代码，方便后续从 Albion 静态数据导出或社区维护数据源更新。

### 市场价格数据

使用 Albion Online Data Project API：

- Asia host：`https://east.albion-online-data.com`
- 当前价格接口：`/api/v2/stats/prices/{item_ids}.json`

默认价格选择：

- 材料成本默认使用 `sell_price_min`，表示直接从最低卖单买入材料。
- 成品收入默认使用 `sell_price_min`，表示按当前最低卖单价格挂卖成品。
- 如果 API 返回字段为空或为 `0`，页面视为无有效数据。

字段选择封装到价格适配器中，并用 mock 响应测试固定行为。后续如果需要“速卖”模式，可以增加 `buy_price_max` 作为成品收入价格来源。

### 图标数据

不批量下载图标到项目。根据物品 ID 动态生成官方渲染图 URL：

```text
https://render.albiononline.com/v1/item/{itemId}.png?quality={quality}&size=217
```

装备、材料列表、搜索结果和计算明细都使用该 URL 显示图标。图标加载失败时显示本地占位图，不影响计算。

### 用户本地数据

保存到 `localStorage`：

- 是否会员。
- 默认制造城市。
- 默认制造站。
- 是否使用专注。
- 返还率覆盖值。
- 最近选择装备。
- 手动价格覆盖。

## 计算规则

第一版按单件装备计算，不做多轮复投模型。

```text
材料毛成本 = Σ(材料数量 × 材料单价) / 单次制造产出数量

返还材料价值 = Σ(材料数量 × 返还率 × 材料单价) / 单次制造产出数量

返还后成本 = 材料毛成本 - 返还材料价值 + 单件制造手续费

销售税率 = 会员 ? 4% : 8%

销售税 = 成品售价 × 销售税率

净收入 = 成品售价 - 销售税

单件利润 = 净收入 - 返还后成本

利润率 = 单件利润 / 返还后成本

保本售价 = 返还后成本 / (1 - 销售税率)
```

销售税率由会员开关决定，用户不能手动输入。第一版只计算销售税，不把挂单设置费做成独立费用项；如果后续需要，可作为单独只读规则项加入。

## 错误处理

- 价格 API 无数据：显示“无市场数据”，价格留空或为 `0`，允许手填。
- 价格过期：展示价格时间；超过设定阈值时用警示状态提示。
- 网络失败：保留已有价格和手动覆盖，提示本次拉取失败。
- 图标加载失败：显示本地占位图。
- 配方缺失：显示“缺少配方，暂不可计算”，不输出利润结果。
- 输入异常：价格为空、负数、返还率超出范围、手续费为空时，结果区显示无法计算的原因。
- 计算边界：返还后成本小于等于 `0`、税率大于等于 `1` 时不计算利润率或保本价。

## 测试策略

- 单元测试覆盖利润公式、会员/非会员销售税、返还率、保本价和非法输入。
- 数据测试检查装备配方的材料 ID、数量、中文名、图标 URL 生成所需字段是否完整。
- API 适配测试使用 mock Asia API 响应，验证市场价字段选择和无数据处理。
- 前端交互测试覆盖搜索装备、拉取价格、手动覆盖价格、切换会员、调整返还率后结果刷新。
- 浏览器验收使用桌面和移动视口检查页面不空白、核心控件可见、文本不重叠、图标可加载或有占位。

- 本地使用。
- 纯前端 Web。
- Asia 服务器。
- 只做装备制造。
- 单个装备计算。
- 自动拉价加手动覆盖。
